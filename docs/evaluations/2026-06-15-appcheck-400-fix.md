# App Check 400 — exchangeRecaptchaV3Token — 2026-06-15

**Status:** löst 2026-06-15 — ny reCAPTCHA v3-site (`livskompassen3.0`), App Check secret i Console, `VITE_APP_CHECK_RECAPTCHA_SITE_KEY` i `.env` + GitHub Actions secret + hosting deploy.

**Symptom:** Console 400 på `exchangeRecaptchaV3Token` · Firestore 95% overifierat  
**Projekt:** `gen-lang-client-0481875058` · Web-app: `1:1084026575972:web:2c85731a56adeb07dbd371`

---

## Orsak

Site key i prod-bygget (`VITE_APP_CHECK_RECAPTCHA_SITE_KEY`) matchar **inte** reCAPTCHA v3-providern registrerad under App Check för web-appen — eller domän saknas i reCAPTCHA.

**App fungerar ändå** tills `APP_CHECK_ENFORCE=true` + Console Enforce.

---

## Fix (3 steg, ~5 min)

### 1. Firebase App Check → Apps → Web

1. [App Check → Apps](https://console.firebase.google.com/project/gen-lang-client-0481875058/appcheck/apps)
2. Välj **web-appen** (Livskompassen)
3. **Registrera** / **Redigera** provider → **reCAPTCHA v3**
4. Klistra in **samma** site key som i `.env`: `VITE_APP_CHECK_RECAPTCHA_SITE_KEY`
   - Måste börja med `6L` (v3) — **inte** secret key (`6L...` site vs `6L...` secret är olika fält)

### 2. Google reCAPTCHA Admin — domäner

1. [reCAPTCHA Admin](https://www.google.com/recaptcha/admin) → samma site key
2. **Domains** ska inkludera:
   - `gen-lang-client-0481875058.firebaseapp.com`
   - `gen-lang-client-0481875058.web.app`
   - `localhost` (lokal dev)

### 3. Prod hosting rebuild

Nyckeln bakas in vid build — efter steg 1–2:

```bash
cd /Users/Livskompassen/StudioProjects/Livskompassen3.0
npm run build
firebase deploy --only hosting
```

Eller push till `main` (CI har `VITE_APP_CHECK_RECAPTCHA_SITE_KEY` secret).

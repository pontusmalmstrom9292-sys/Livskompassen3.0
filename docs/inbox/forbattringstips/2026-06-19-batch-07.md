## 1. Smart Inkast Supermodul

### Arkitektur och Ansvar

- **UI:** Komponenter för multimodal upload (text, bild, röst) integrerat i ChameleonInputShell.
- **State:** Lokal state hanteras av Zustand; offline cache i IndexedDB via Workbox.
- **Backend:** Cloud Functions för hantering av uploads, AI-klassificering (Vertex AI), event-triggers i Firestore för pipeline.
- **AI:** API-anrop till Vertex AI för embedding och klassificering, också för emotional analysis.

### Dataflöden

1. Användarinput i frontend laddas upp (text/bild/röst).  
2. Indexeras i lokal IndexedDB vid offline, eller skickas direkt till backend vid online.  
3. Cloud Function `submitInkastLite` tar emot, sparar metadata + media i Firebase Storage/Firestore.  
4. Triggar AI-klassificering och sensible pipeline-funktioner.  
5. Resultat (klassificering/feedback) skrivs tillbaka till Firestore och skickas till frontend.  
6. UI anpassar onboarding och prioriterar synkning baserat på AI-feedback.

### Nyckelprinciper

- Robust offline-first UX.  
- AI-driven prioritering av synkning.  
- Flexibel upload modalitet.  
- Låg latens i klassificeringsfeedback.

## 2. Adaptiv Onboarding Supermodul

### Arkitektur och Ansvar

- **UI:** React Joyride som guide, chameleon-teman som ändrar karaktär (calm, neutral, energetic).  
- **State:** Zustand store sparar progress, loggar för analys.  
- **Backend:** Loggning via callables, AI-hjälp via Vertex AI Chat integration.  
- **AI:** Kontextkänslig dialog med Vertex AI Chat, promptbaserad hjälp på varje onboarding-steg.

### Dataflöden

1. Onboarding startas vid första inkast eller vid ny funktion.  
2. Joyride steg visas dynamiskt baserat på user-state och AI-feedback.  
3. AI Chat hjälper vid problem eller frågor, anpassar svar till användarens beteende.  
4. Progress sparas i Firestore för fortsatt analys och iteration.  

### Nyckelprinciper

- Microlearning i små steg.  
- AI-driven Anpassning efter användarens beteende och behov.  
- Smidig integrering i ChameleonInputShell för tematisk konsistens.

## 3. Valv Supermodul (Högsäker Lagring)

### Arkitektur och Ansvar

- **UI:** Säker inloggning med WebAuthn/biometri i chameleon-skal.  
- **Databas:** Append-only, WORM-princip för känslig data i Firestore.  
- **Backend:** Rollbaserad åtkomstkontroll via RBAC och Firebase Auth custom claims.  
- **Säkerhet:** App Check enforcement, rate-limiting och strikt datavalidering.

### Dataflöden

1. Användare loggar in säkert med WebAuth.  
2. Valv-data skrivs som append-only, utan möjlighet att ändra bort tidigare poster.  
3. AI-agenter övervakar ovanlig aktivitet och potentiella attacker.  
4. Backup och återställningsprocedurer körs automatiskt.  

### Nyckelprinciper

- Oemotsäglig dataintegritet (WORM).  
- Säkerhet på flera nivåer.  
- Automatisk övervakning och reaktion.

## 4. Familjen Supermodul (Barnfokus & Peer Sync)

### Arkitektur och Ansvar

- **UI:** Barnfokuserad PWA med barnanpassad UX och känslobaserad feedback.  
- **Backend:** Separat silo med egen synk/orkestrering via Firebase och WebRTC peer-to-peer.  
- **Data:** Offline-first med intelligent sync när möjlighet finns.  
- **AI:** Emotionell analys och trygghetsindikatorer i realtid.

### Dataflöden

1. Barn och föräldrar interagerar via PWA, data cachas offline.  
2. Peer-to-peer-synk mellan närliggande enheter via WebRTC.  
3. AI analyserar emotionssignaler och ger stöttande råd.  
4. All data säkerställs i egen silo med strikt accesskontroll.

### Nyckelprinciper

- Säker och rolig barnanpassad UX.  
- Lokal peer sync utan internet.  
- AI-driven trygghetsstöd.

## 5. Kompass Supermodul (Adaptiv Startsida)

### Arkitektur och Ansvar

- **UI:** Dynamisk dashboard som anpassar widgets och innehåll baserat på kapacitet och AI-insikt.  
- **State:** Styrs via Zustand och remote config för snabb anpassning.  
- **Backend:** Insikter genereras via AI-baserade micro-agenter och WORM-data.  

### Dataflöden

1. Användardata aggregeras och analyseras i bakgrunden.  
2. AI genererar prioriterade insikter och presenterar i kompassens UI.  
3. Dashboard adapteras i realtid efter användarens kapacitet, belasta ej vid stress.  

### Nyckelprinciper

- Kapacitetsstyrd adaptiv.  
- AI-genererade prioriteringar.  
- Lågstressad visualisering.

## 6. Ekonomi Supermodul (Budget & AI)

### Arkitektur och Ansvar

- **UI:** Modul för budget, impulsköp och sparmål med AI-förslag.  
- **Backend:** Beräkningar av smart allocation, AI-driven trendanalys.  
- **State:** Hantering av ekonomi-data med optimerad cache.  

### Dataflöden

1. Användare loggar ekonomiaktivitet.  
2. AI identifierar mönster och föreslår budgetallokeringar.  
3. Visual anpassas i chameleon-tema efter användarens beteende och mål.  

### Nyckelprinciper

- AI-stött ekonomistyrning.  
- Personlig anpassning.  
- Modulär och datalagring.
